# Onboarding

## Start Here

1. Read `README.md`.
2. Read `vision/mission.md`.
3. Read `projects/wealthlens-dashboard/CONTRIBUTING.md`.
4. Pick a small task from `tasks/backlog.md` or a labelled issue once GitHub Issues are set up.

## Contributor Expectations

- Be specific.
- Cite data sources.
- Keep accessibility in mind.
- Ask for clarification when methodology is unclear.
- Prefer small pull requests.
